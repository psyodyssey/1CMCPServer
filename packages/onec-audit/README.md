# onec-audit

Библиотека аудита действий агента в рамках 1C Agent Platform.
На этом шаге пакет перестаёт быть чистым форматтером: появилось
минимальное append-only JSONL-хранилище.

## Что внутри

- `AuditRecord` — структура одной записи аудита (`operation_id`,
  `tool_name`, `environment`, `base_id`, `status`, `message`).
  Контракт не меняется.
- `format_audit_record(record)` — каноническая сериализация
  записи в однострочный JSON (`json.dumps`, `ensure_ascii=False`).
  Формат строки в store задаётся именно этой функцией.
- `append_record(audit_dir, record)` — append-only запись одной
  строки в `<audit_dir>/audit.jsonl`. Каталог создаётся, если его
  нет. Возвращает строковый путь к файлу аудита. Ошибки файловой
  системы (`OSError`) не проглатываются.
- `read_last_record(audit_dir)` — возвращает последнюю непустую
  `AuditRecord` из `<audit_dir>/audit.jsonl`, либо `None`, если
  файл отсутствует (или не содержит ни одной непустой строки).
  На повреждённой JSON-строке или рассогласовании с формой
  `AuditRecord` поднимает `ValueError`; `OSError` пробрасывается
  как есть.

## Что не делает

- Ротация, сжатие и ретенция лога.
- Поиск/фильтрация по журналу — только `read_last_record`.
- Любые write-операции, кроме append.
- Маскирование секретов — полагается на вызывающий код.

Имя файла аудита фиксированное: `audit.jsonl`. Формат — JSONL:
по одной JSON-строке на запись, UTF-8, переводы строк `\n`.

# onec-troubleshooting

Библиотека диагностики и устранения неисправностей 1С.

Здесь живёт логика интерпретации результатов health-проверок: по списку
статусов, собранных `onec-health`, формируется структурированный отчёт
с вероятной причиной и рекомендованным действием. В перспективе сюда
добавятся более глубокие сценарии (разбор ошибок, анализ логов, подсказки).

Сейчас в пакете заведены минимальные контракты:

- `TroubleshootingReport` — отчёт по одной проблеме
  (`problem_code`, `probable_cause`, `recommended_action`);
- `diagnose_from_health(statuses)` — базовое сопоставление известных кодов
  (`gateway_down`, `dump_missing`, `index_lock`) с типовыми выводами;
  для неизвестного набора возвращается `unknown` + рекомендация запустить
  расширенную диагностику.

Это skeleton-уровень: правил пока немного, реальной диагностики процессов,
портов или индексов здесь нет — только контракты и rule-based flow.
